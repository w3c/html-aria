ARIA in HTML - test cases for body, html and head elements

last updated 13 March 2021

Tests:

Test 1

No role is allowed on the body, html or head elements. To manually verify, check this page with a conformance checker. The body element has a role=document, and the head and html elements have role=presentation.

To verify additional roles, use the text field below to enter valid roles and dynamically apply them to the body, html and head elements, and then re-run the conformance checkers.

Warning: adding certain ARIA roles to the body and html elements may make this page inaccessible.

Expectation: conformance checkers will indicate that role=document is redundant on body element, and that a role with any other value is invalid on the body element.

Conformance checkers will indicate that any role is invalid on the html and head elements.
